# chie

<p align="center"><b>chie</b> — убийца nano с душой, заточенной под UniLand.</p>

Терминальный текстовый редактор в духе nano: те же привычки и удобство, но с
настоящей подсветкой синтаксиса — в том числе **своей, ручной подсветкой для
[UniLand](https://codeberg.org/moyunni/uniland)**, а ещё Python и десятка других
языков. Один статический бинарник, ставится в систему, запускается из любого
места. Linux, macOS, Windows.

## Почему chie

nano хорош ровно до того момента, как открываешь код. chie берёт всё, за что
любят nano, и добавляет то, чего ему не хватает:

- **Подсветка UniLand из коробки.** Не через сторонний движок — chie понимает
  ключевые слова, встроенные функции, шаблонные строки `` `…{expr}…` `` и даже
  вложенные блоки `python { … }` внутри `.uni`-файла. Плюс Python, Rust, C/C++,
  JS/TS, Go, JSON, Shell и другие.
- **Настраивается как Ghostty.** Свои цвета (hex/имена/палитра), поведение,
  тема — всё в одном простом конфиге `key = value`. Ничего не нужно
  перекомпилировать.
- **Умные вещи по мелочи:** отмену/повтор с группировкой, поиск с «умным
  регистром», выделение через Shift и мышью, отступ/анти-отступ по всему
  выделению, автоотступ, подсветка строки под курсором, `scrolloff`,
  срезание хвостовых пробелов при сохранении.
- **Не теряет данные.** Сохранение атомарное (через temp-файл и `rename`),
  с опциональными бэкапами — файл не обрежется, даже если что-то пойдёт не так.
- **Мышь и Unicode.** Клик/протяжка/колесо; корректная ширина CJK и эмодзи —
  курсор не разъезжается.
- **Один бинарник.** Ни рантайма, ни зависимостей у пользователя — собрал и
  положил в `PATH`. Linux, macOS, Windows.
- **Привычки nano сохранены.** `^G`, `^O`, `^W`, `^K`, `^X` — всё на своих местах.

## Установка

Нужен [Rust](https://rustup.rs) 1.74+ (для сборки; пользователю рантайм не нужен).

### Через cargo (проще всего)

```bash
cargo install --path .        # из клонированного репозитория
```

Бинарник окажется в `~/.cargo/bin/chie` — эта папка обычно уже в `PATH`.

### Скриптом установки

```bash
bash install.sh                                   # Linux / macOS → ~/.local/bin/chie
powershell -ExecutionPolicy Bypass -File install.ps1   # Windows
```

Скрипт соберёт релизную версию и положит бинарник в пользовательский `bin`.

### Вручную

```bash
cargo build --release
install -m755 target/release/chie ~/.local/bin/chie   # или куда удобно
```

Проверка:

```bash
chie --version
```

## Быстрый старт

```bash
chie file.uni        # открыть (или создать) файл
chie                 # пустой буфер
```

Дальше как в nano: пишешь, а внизу всегда видны основные бинды. Нажми **`^G`**
(Ctrl+G) — откроется полный список. Нажми **`Alt+A`** — узнаешь, что это за
программа и где её репозитории.

Выход — **`^X`** (Ctrl+X). Если есть несохранённые правки, chie спросит,
сохранять ли.

## Аргументы командной строки

```
chie [ОПЦИИ] [ФАЙЛ]

  -h, --help            показать помощь
  -v, --version         версия
      --config          показать путь к конфигу и выйти
      --tabsize <N>     ширина отступа в пробелах (по умолчанию 4)
      --tabs            делать отступ настоящими табами, а не пробелами
      --no-syntax       запуститься без подсветки
      --no-numbers      запуститься без номеров строк
      --no-autoindent   не переносить отступ на новую строку
      --no-mouse        не захватывать мышь
```

Подсветку и номера строк можно переключать и на ходу — `Alt+Y` и `Alt+N`.

## Полный список биндов

Обозначения: `^X` — Ctrl+X, `M-X` — Alt+X (Meta), `S-` — Shift.

### Перемещение

| Бинд | Действие |
|------|----------|
| `←` `→` `↑` `↓` | по символам и строкам |
| `^←` `^→` / `M-←` `M-→` | на слово влево / вправо |
| `Home` / `^A` | умное начало строки (сначала к первому непробелу, потом к колонке 0) |
| `End` / `^E` | конец строки |
| `PageUp` / `PageDown` | на экран вверх / вниз |
| `M-V` / `^V` | на экран вверх / вниз (эмакс-стиль) |
| `^Home` | в начало файла |
| `^End` | в конец файла |
| `M-\` | к первой строке |
| `M-/` | к последней строке |
| `S-` + любое перемещение | расширить выделение |

### Правка

| Бинд | Действие |
|------|----------|
| `Tab` / `S-Tab` | отступ / анти-отступ (работает на всём выделении) |
| `Enter` | новая строка с автоотступом |
| `Backspace` | удалить символ слева |
| `Delete` / `^D` | удалить символ справа |
| `^K` | вырезать выделение, а без него — строку (подряд накапливаются) |
| `M-6` / `^C` | скопировать выделение или строку |
| `^U` | вставить |
| `^Z` / `M-U` | отменить |
| `M-E` / `^Y` | повторить |
| `M-M` | поставить / снять метку выделения |

### Файлы

| Бинд | Действие |
|------|----------|
| `^O` | записать файл (спросит имя) |
| `^S` | сохранить |
| `^R` | вставить содержимое другого файла в позицию курсора |
| `^X` | выход (предложит сохранить) |

### Поиск

| Бинд | Действие |
|------|----------|
| `^W` | найти (умный регистр: строчными — без учёта регистра) |
| `M-W` | найти следующее |
| `^\` | найти и заменить (по одному: `Y`/`N`/`A`=все/`Esc`) |
| `M-G` / `^_` | перейти к строке или `строка,колонка` |

### Вид и помощь

| Бинд | Действие |
|------|----------|
| `M-N` | номера строк вкл / выкл |
| `M-Y` | подсветка вкл / выкл |
| `^L` | центрировать по курсору / перерисовать |
| `^G` / `F1` | помощь (этот список внутри редактора) |
| `M-A` | **о программе:** автор, репозитории, телеграм |

> В нижней части экрана всегда видны две строки с самыми частыми биндами —
> подсматривать в этот список каждый раз не придётся.

## Мышь

Из коробки (можно выключить — `--no-mouse` или `mouse = false` в конфиге):

- **клик** — поставить курсор;
- **протяжка** — выделить текст;
- **колесо** — прокрутка.

Пока мышь захвачена, обычное выделение терминала не работает — это нормально для
TUI-редакторов. Нужен системный копипаст мышью — отключи мышь или подержи Shift
(зависит от терминала).

## Подсветка синтаксиса

Движок подсветки написан вручную, без внешних библиотек. Он работает построчно
и «протаскивает» состояние между строками, поэтому многострочные конструкции —
блочные комментарии, тройные строки Python, шаблонные строки UniLand и целые
блоки `python { … }` — раскрашиваются правильно.

### UniLand

Для `.uni` chie различает и раскрашивает:

- **структурные ключевые слова** — `let`, `const`, `func`, `if`, `for`, `while`,
  `return`, `try`/`catch`, `throw`, `python`, …;
- **константы** — `true`, `false`, `null`;
- **встроенные функции** — весь стандартный набор (`print`, `map`, `read_file`,
  `http_get`, `gui_window`, …);
- **строки** `"…"` с экранированием;
- **шаблонные строки** `` `…` `` с интерполяцией `{expr}` (и экранированием
  `{{` / `}}`);
- **числа**, **операторы**, три вида комментариев — `//`, `#`, `/* … */`;
- **вложенный Python.** Внутри блока
  `python { … }` код раскрашивается уже по правилам Python — вплоть до парной
  закрывающей скобки, даже если блок занимает много строк.

```uniland
// шаблоны, интерполяция и встроенный питон — всё подсвечено
let name = "мир"
print(`Привет, {name}!`)

python {
    import math
    uni.phi = (1 + math.sqrt(5)) / 2
}
```

### Остальные языки

Определение — по расширению, по известному имени файла (`Makefile`,
`Dockerfile`, `Cargo.toml`) или по shebang.

| Язык | Расширения |
|------|-----------|
| Python | `.py` `.pyw` `.pyi` |
| Rust | `.rs` |
| C / C++ | `.c` `.h` `.cpp` `.cc` `.hpp` … |
| JavaScript / TypeScript | `.js` `.ts` `.jsx` `.tsx` `.mjs` … |
| Go | `.go` |
| JSON | `.json` |
| Shell | `.sh` `.bash` `.zsh` |
| TOML | `.toml` |
| YAML | `.yml` `.yaml` |
| HTML / XML | `.html` `.htm` `.xml` |
| CSS / SCSS | `.css` `.scss` |
| Lua | `.lua` |
| Makefile, Dockerfile | по имени |

Незнакомый файл открывается без подсветки — это честнее, чем красить наугад.
Добавить новый язык — дело нескольких строк в `src/highlight/generic.rs`.

## Настройка (config)

chie читает `~/.config/chie/config` (путь можно узнать командой `chie --config`;
на Windows — `%APPDATA%\chie\config`). Формат — как у Ghostty: по одной строке
`ключ = значение`, `#` в начале строки — комментарий. Незнакомые ключи не роняют
редактор, а показываются предупреждением при запуске.

Порядок приоритета: **встроенные умолчания → файл конфига → флаги командной
строки**.

Логические значения принимают `true/false`, `on/off`, `yes/no`, `1/0`.

**Поведение:**

```ini
# отступы
tabsize = 4           # ширина отступа (1..16)
tabs = false          # отступ настоящими табами вместо пробелов
autoindent = true     # переносить отступ на новую строку

# вид
numbers = true        # номера строк
syntax = true         # подсветка при старте
cursorline = false    # подсвечивать строку под курсором
scrolloff = 0         # держать N строк запаса сверху/снизу от курсора (0..99)
mouse = true          # захват мыши

# сохранение
backup = false        # оставлять копию "<файл>~"
trim_trailing = false # срезать хвостовые пробелы при сохранении
final_newline = false # гарантировать перевод строки в конце файла

# поиск
ignorecase = true     # поиск без учёта регистра
smartcase = true      # но с учётом, если в запросе есть заглавная (как в vim)
```

**Свои цвета (темизация).** Значение — `<цвет> [bold]`, где цвет это имя
(`red`, `blue`, `brightmagenta`, …), hex `#rrggbb`, индекс палитры `0..255`
или `default`:

```ini
# 15 классов синтаксиса:
color.keyword  = brightmagenta bold
color.string   = #8fd98f
color.comment  = 244
color.number   = brightcyan
# ... builtin, type, function, template, interp, constant, operator,
#     punct, escape, preproc, text

# хром редактора:
color.bar_bg     = white     # верхняя/нижняя панели
color.key_bg     = blue      # подсветка биндов в подсказке
color.selection  = blue      # фон выделения
color.match_bg   = yellow    # фон найденного при поиске
color.cursorline = 236       # фон строки под курсором
color.gutter     = brightblack
```

Полный список ключей с комментариями — в файле [`config.example`](config.example).

## Удаление

chie — это один бинарник. Как удалять — зависит от того, как ставил:

| Как ставил | Как удалить |
|---|---|
| `cargo install` | `cargo uninstall chie` |
| `install.sh` | `rm ~/.local/bin/chie` |
| вручную | `rm` там, куда клал |
| Windows / `install.ps1` | `del %USERPROFILE%\.local\bin\chie.exe` |
| исходники | `rm -rf ~/chie` |

Если делал себе конфиг — убери и его: `rm -rf ~/.config/chie`
(на Windows — `%APPDATA%\chie`). Больше chie нигде ничего не хранит.

## Устройство проекта

```
src/
  main.rs            разбор аргументов, точка входа, тесты
  editor.rs          состояние редактора, цикл событий, отрисовка, бинды, мышь
  buffer.rs          хранение текста и отмена/повтор
  config.rs          чтение ~/.config/chie/config (поведение + тема)
  theme.rs           семантические стили → цвета терминала, разбор цветов
  width.rs           ширина символов в терминале (CJK/эмодзи/комбинирующие)
  about.rs           данные для экрана «о программе»
  highlight/
    mod.rs           тип Style, трейт Highlighter, определение языка
    uniland.rs       ручная подсветка UniLand (+ вложенный python)
    python.rs        подсветка Python (переиспользуется в блоках .uni)
    generic.rs       табличная подсветка остальных языков
```

Подсветка говорит только семантическими классами (`Keyword`, `String`, `Number`,
…), а превращает их в цвета `theme.rs`. Хочешь свою тему — правишь один файл.

## Сборка и разработка

```bash
cargo build            # отладочная сборка
cargo build --release  # релиз (один оптимизированный бинарник)
cargo test             # тесты буфера, подсветчиков, конфига (TTY не нужен)
cargo clippy           # линты
cargo fmt              # форматирование
```

**Платформы.** Linux, macOS, Windows — код кроссплатформенный через `crossterm`.
CI (`.github/workflows/ci.yml`, читается и Forgejo/Gitea-раннерами) на каждый
пуш собирает, тестирует и линтует проект на всех трёх ОС.

Форкаешь? Пара слов про стиль: комментарии тут пишутся как для человека — коротко
и по делу, объясняют «почему», а не пересказывают код. Если добавляешь язык или
бинд — держись этого тона.
